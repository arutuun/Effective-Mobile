Блок-схема процесса отправки PUSH-уведомлений:

1. Мобильное приложение запускает процесс:  
* Запрашивает у пользователя разрешение на уведомления.  
* Если разрешение получено \- получает токен от сервиса уведомлений (APNs/FCM).  
* Отправляет токен на бэкенд-систему.

2. Бэкенд-система сохраняет токен в базу данных:  
* При наступлении события (например, новая акция) формирует уведомление.  
* Отправляет уведомление во внешнюю систему (APNs/FCM).

3. Внешняя система (сервисы APNs для iOS и FCM для Android):  
* Доставляет уведомление на устройство пользователя;  
* Возвращает статус доставки обратно на бэкенд.

Итог: пользователь получает PUSH-уведомление на смартфон. Процесс состоит из трёх частей: мобильное приложение → бэкенд-система → внешняя система (сервисы уведомлений).
